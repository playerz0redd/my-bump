# my-bump

Курсовой проект для БГУИР представляет собой мобильное приложение для iOS, являющееся аналогом сервиса Zenly. Основная идея приложения — предоставление пользователям возможности интерактивного взаимодействия на карте, обмена геопозицией в реальном времени, управления списком друзей и общения в чатах с соблюдением настроек приватности.

---

## Технологический стек
* Swift, SwiftUI, UIKit, Combine, MapKit, CoreLocation, Firebase Auth, Firebase Firestore, Firebase Realtime Database, Google Sign-In, Cloudinary, Kingfisher, Swift Package Manager.

* **Пользовательский интерфейс:** Смешанный подход. Основной интерфейс написан на **SwiftUI**. Списки чатов реализованы на **UIKit** (`UICollectionView` с `UICollectionViewDiffableDataSource`).
* **Реактивное программирование:** **Combine** (применяется для связи между сервисами, менеджерами геопозиции и презентационным слоем. Все экраны приложения построены с использованием Combine, непрерывно в реальном времени слушают обновления из Firestore и мгновенно изменяют свое состояние).
* **Бэкенд и База данных:**
  * **Firebase Auth** — регистрация, авторизация (Email/Password и интеграция Google Sign-In).
  * **Firebase Firestore** — хранение профилей пользователей, структуры социальных связей (запросы в друзья, статусы отношений) и истории сообщений в чатах.
  * **Firebase Realtime Database** — для синхронизации координат пользователей в реальном времени.
* **Работа с изображениями:**
  * **Cloudinary** — облачное хранилище для загрузки аватаров пользователей и оптимизации их размера с помощью трансформаций.
  * **Kingfisher** — кэширование и асинхронное отображение изображений в интерфейсе.
* **Карты:** **MapKit**.
* **Менеджер зависимостей:** **Swift Package Manager (SPM)**.

## Архитектура приложения

Приложение построено с разделением ответственности на основе слоев абстракции, сочетая принципы **Clean Architecture** и паттерн **MVVM-C** (Model-View-ViewModel-Coordinator).

### Ключевые компоненты архитектуры:

* **Core / Managers** — низкоуровневые слои работы с системными службами и внешними сервисами.
* **Services** — слой бизнес-логики.
* **Presentation Layer** — слой представления:
  * **ViewModels:** Используют макрос `@Observable` (для SwiftUI) и `@Published` / `ObservableObject` (для интеграции с UIKit), изолированы на `MainActor`. Они управляют состоянием экранов и обрабатывают действия пользователей.
  * **Views:** Разделены по фичам. Используют кастомные компоненты и модификаторы (`ImageSelectorModifier`, `InputField`, `AuthButton`).
* **Dependency Injection:**
  * `AppDependencyContainer` выступает единой точкой конфигурации зависимостей. Он инициализирует менеджеры и предоставляет фабричные методы (`ProfileScreenFactory`, `MapScreenFactory` и др.) для сборки экранов и их ViewModel.

## Функциональность

### Регистрация и авторизация:
* Создание учетной записи с валидацией имени, почты и пароля.
* Авторизация через Google Sign-In.
* Сброс забытого пароля с отправкой ссылки на почту и интерактивным экраном подтверждения.

### Геолокация и социальное взаимодействие:
* Фоновое отслеживание координат текущего пользователя.
* Отображение местоположения друзей на карте в реальном времени.
* Кастомные маркеры друзей с их аватарами и именами. Маркеры анимированы с помощью ключевых кадров (`KeyframeAnimator`) для создания эффекта движения.
* Поиск пользователей по имени.
* Отправка запросов в друзья.
* Разделы входящих запросов и списка текущих друзей с возможностью быстрого принятия или отклонения.
* Возможность удаления пользователей из друзей.

### Чат в реальном времени:
* Отображение списка чатов с превью последнего сообщения, временем его получения и счетчиком непрочитанных.
* Мгновенный обмен текстовыми сообщениями внутри диалога через подписку на Firestore snapshot-слушатели.
* Автоматический скролл и бабблы сообщений (`MessageBubble`), меняющие ориентацию в зависимости от отправителя.

### Управление профилем:
* Загрузка аватара напрямую с камеры устройства или из медиатеки. Автоматическое обновление аватара у всех друзей в их списках контактов.

## Реализованные решения

### 1. Гибридная база данных (Firestore + Realtime Database)
Использование баз данных разделено по характеру нагрузки:
* Изменения координат происходят часто. Для минимизации задержек и стоимости запросов геопозиция записывается в древовидную структуру Realtime Database. Обновление координат оптимизировано с помощью оператора `throttle` в Combine, ограничивающего частоту отправки до одного раза в 5 секунд.
* Сообщения, профили и списки друзей требуют сложных выборок и связей, поэтому хранятся в документах и коллекциях Firestore. Все экраны взаимодействуют с базой данных реактивно, непрерывно отслеживая изменения.

### 2. Производительность списков в UIKit внутри SwiftUI
Для экрана списка чатов (`ChatFlowView`) используется обертка над `UIViewController` — `ChatsViewController`. Внутри него инициализирован `UICollectionView` с механизмом `UICollectionViewDiffableDataSource` для обеспечения плавной работы интерфейса при частых обновлениях данных.

### 3. Безопасная конкурентность при работе с коллекциями
Для ускорения сетевых запросов написаны асинхронные расширения для коллекций:
* `asyncMap`
* `concurrentMap`

### 4. Динамические кастомные формы и анимации
* Реализована кастомная структура `MessageBubble: Shape` для отрисовки баббла сообщения со скруглением только определенных углов.
* На экране восстановления пароля применяется `PhaseAnimator`, плавно переключающий системные иконки по кругу, визуализируя процесс отправки письма.
